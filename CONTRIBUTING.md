# Contributing to Prescription Learning Platform\n\nWe love your input! We want to make contributing to this project as easy and transparent as possible.\n\n## Development Setup\n\n1. Fork the repository\n2. Clone your fork\n3. Create a feature branch (`git checkout -b feature/AmazingFeature`)\n4. Make your changes\n5. Commit your changes (`git commit -m 'Add some AmazingFeature'`)\n6. Push to the branch (`git push origin feature/AmazingFeature`)\n7. Open a Pull Request\n\n## Code Style\n\n- Use meaningful variable names\n- Add comments for complex logic\n- Follow PEP 8 for Python\n- Follow ESLint rules for JavaScript\n\n## Reporting Bugs\n\n- Use a clear descriptive title\n- Describe the exact steps to reproduce\n- Provide specific examples\n- Describe the observed behavior\n- Explain the expected behavior\n\n## Feature Requests\n\n- Use a clear descriptive title\n- Provide a step-by-step description\n- Provide specific examples\n- Describe the current behavior\n- Describe the desired behavior\n
